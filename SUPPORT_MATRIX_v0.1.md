# Camelid v0.1 Support Matrix

Status date: 2026-05-31

This is the v0.1 release-candidate support boundary. It is stricter than the broader repo docs where those docs conflict with evidence manifests.

Public support is exact-row only: model file, model family, tokenizer/template path, quantization, API/frontend support contract, context bucket, and cited evidence path all have to match. Neighboring sizes, variants, quants, templates, hosts, or contexts do not inherit support.

## Minimum Evidence to Promote Any Unsupported Row

Promotion requires a new row-specific durable bundle with:

- exact model filename, row id, SHA256, source/runtime head, and clean checkout state
- prompt-token parity against the known-good reference
- generated-token and generated-text parity for the stated prompt pack and token budget
- API load, `/v1/completions`, `/v1/chat/completions`, `/api/capabilities`, and frontend smoke where user-facing support is claimed
- explicit support contract fields, including compatibility row/status, `contract_supported`, `generation_ready`, `loaded_now`, and WebUI chat state
- RSS/timing boundaries when memory or performance is mentioned
- `SHA256SUMS` and public-scrub/privacy validation
- a claim boundary that says what is not promoted
